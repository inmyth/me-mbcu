Portfolio

Market making bots

Market making is a trading technique that places buy and sell orders simultaneously and profits from bid-offer spread. In short, buy low sell high. Automating the trades is the job for bot which requires speed and robustness. There are two bots developed for different protocols.

For Ripple network

Ripple blockchain allows trades of arbitrary assets. As such it can be deployed as off-the-shelves crypto exchange. This bot was built on behalf of such exchange to provide liquidity for its users to trade. It maintains in-memory states which are asynchronously modified by trades, cancellations, and other actions. Source at Github

  • Coded in Java
  • Worked with RxJava and synchronized blocks for concurrency
  • Operates on Ripple network with websocket
  • Features error failsafes to keep the bot running uninterrupted

For Private exchanges

Similar to the first one but operates on private exchanges such as HitBTC, Okex, etc. As Java's synchronized was not sufficient for modern concurrency work, Scala and Akka was adopted. The move cut the development time to a few weeks. Source at Github

  • Coded in Scala
  • Adopting Scala's functional programming resulted in reduced code base and complexity
  • Utilized Akka Actor for concurrency work

The video below illustrates the bot's account during operation on Ripple Net. It starts from seeding to countering trades and replacing consumed offers.

Farming prediction app

For farmers, maximizing profit isn't the same as maximizing yield. Although crops are restricted by season, there's a variation in timing influenced by demand that may result in better market price. This project was to predict potential opportunities (as well as losses) using mathematical tools and machine learning for agriculture region in Kyushu, Japan. It was supported by IBM and eventually acquired and co-developed with Softbank. Press release by Japan Ministry of Agriculture and Fishery.

  • Built with Javascript, MySQL
  • Worked with IBM Bluemix cloud stacks

Home monitoring app

An app to manipulate home appliances as well as monitoring the surroundings, developed before IOT became a word. It features bi-directional stream capability between camera and Android devices essentially turning the system into a video telephone.

  • Android 2.1
  • Implemented basic Voice over Internet Protocols (VOIP) for audio communication
  • Worked with MP4 and MJPEG for video stream

If youâ€™re still using Synchronized, you should try Akka Actor instead

In the Java world, synchronized is the conventional way of doing concurrency. Looking in retrospect, it's probably one reason why people were scared of multi-threading. Synchronized is basically a locking mechanism to prevent multiple threads from modifying a state at the same time, a scenario knows as race condition. The problem is managing the lock is massively difficult. Actor is a concurrency model that solves this issue by lining up all inputs in a queue. As the messages are digested one-by-one the state can be altererd safely without the need for locks. Read the rest at FreeCodeCamp.
